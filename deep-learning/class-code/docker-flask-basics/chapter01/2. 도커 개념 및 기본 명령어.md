# 1. Docker란?
: Docker 는 애플리케이션과 그 종속성을 격리된 환경인 컨테이너로 패키징하여 OS 에서 독집적인 오픈소스 가상화 플랫폼. Go 언어로 작성되어 리눅스 컨테이너를 기반으로 하며 개발자가 어플케이션을 쉽고 빠르게 배포할 수 있도록 설계됨

# 2. 가상머신 (VM) 과 도커 (Docker) 컨테이너의 차이점
: Docker 는 기존 가상머진 (Virtual Machine) 의 한계를 극복하고 효율적인 환경을 제공
**리소스 효율성:** 가상머신은 각각 완전한 게스트 OS(운영체제)를 포함해야 하지만, 도커 컨테이너는 호스트 OS의 커널을 공유하므로 훨씬 가볍고 빠르게 실행
 **격리 수준:** 컨테이너는 호스트 OS의 리소스를 논리적으로 분리하여 사용하며, 각각의 컨테이너는 독립적으로 격리된 상태로 동작
**실행 속도:** 별도의 무거운 OS를 부팅할 필요가 없기 때문에 컨테이너는 거의 즉시 실행
**오버헤드 감소:** 가상머신은 하이퍼바이저와 게스트 OS 구동으로 인해 오버헤드가 크지만, 컨테이너는 이러한 시스템 부담(오버헤드)이 현저히 적음

# 3. 도커 핵심 구성 요소
- **Docker Engine** : 컨테이너를 실행하고 관리하는 주체로 도커의 핵심 기술로 다양한 Linux 배포판과 Window에서 컨테이너를 지원하는 클라이언트-서버 애플리케이션.
- **Docker Image** : 컨테이너로 실행될 소프트웨어 모음으로 애플리케이션 코드, 런타임, 라이브러리, 시스템 도구 등 실행에 필요한 모든것을 포함. 이미지는 변경 불가하고 수정이 필요한 경우 새 이미지를 생성해야함
- **Docker Container** : 실행중인 도커 이미지의 인스턴스로 각 컨테이너는 자체 파일 시스템, 네트워크 스택, 프로세스 공간을 가지며 CPU 와 메모리 리소스 제한이 정의되어있음
- **Docker Registry** : 이미지를 저장하고 배포하는 저장소로, Docker Hub 가 대표적인 공개 레지스트리

# 4. Docker 장점
- **일관된 환경** : 어떤 컴퓨터로 이식해도 개발, 테스트, 프로덕션 환경 간의 일관성 보장
- **빠른 배포** : 컨테이너틑 가볍고 빠르게 시작되므로 애플리케이션 배포 시간 단축
- **확장성** : 컨테이너는 쉽게 복제하고 확장 할 수 있어 마이크로서비스 아키텍처에 적합
- **이식성** : 컨테이너화된 애플리케이션은 어떤 환경에서도 동일하게 실행
- **격리** : 각 컨테이너는 독립적으로 실행되어 보안과 리소스 관리 측면에서 이점 제공

# 2. 도커 기본 명령어
- `docker pull`: 도커 이미지를 다운로드함 (예: `docker pull httpd`)
-  `docker images`: 다운로드된 이미지의 리포지토리, 태그, 이미지 ID, 생성일, 크기 등을 확인함
- `docker run`: 이미지를 기반으로 컨테이너를 실행함
  * 주요 옵션: `-d`(백그라운드 실행), `-p`(포트 매핑), `--name`(컨테이너 이름 지정), `-it`(대화형 터미널 사용), `-v`(볼륨 마운트/폴더 공유)
- `docker ps`: 실행 중인 컨테이너 상태(ID, 이미지, 명령어, 생성일, 포트 등)를 확인함

# 3. 파이썬 베이스 이미지 특징
- `python:X.Y`: 표준 Debian 기반. 패키지가 다양하여 안정적이고 확장성이 높지만 용량이 큼
- `python:X.Y-slim`: 불필요한 패키지를 제거하여 용량을 최소화함. 빌드 시 필요한 패키지는 직접 설치해야 함 
- `python:X.Y-alpine`: Alpine Linux 기반으로 초경량이나, 호환성 문제 및 일부 패키지 빌드에 어려움이 있을 수 있음
