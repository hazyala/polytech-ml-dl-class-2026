# 1. Flask란 무엇인가? 
- **정의** : 파이썬으로 작성된 마이크로 웹 프레임워크
> 마이크로란? 웹 서버 운영에 꼭 필요한 핵심 기능 (라우팅, 요청/응답 처리, 템플릿 렌더링 등) 만 기본으로 제공하고, 나머지 기능 (데이터베이스, 인증 등)은 필요에 따라 확장 할 수 있다는 의미
- **확장성** : 데이터베이스 연동이나 사용자 인증 같은 부가 기능은 사용자가 필요할 때마다 라이브러리를 추가하여 자유롭게 확장 가능

# 2. Flask의 장점
- **학습 용이성** : 구조가 단순하여 배우기 쉬움
- **유연성 및 확장성** : 초보자의 간단한 토이 프로젝트부터 전문가의 복잡한 애플리케이션까지 널리 사용

# 3. Flask 핵심 구성 요소
: Flask는 자체적으로 모든 것을 개발하기 보다 검증된 두 가지 라이브러리를 기반으로 돌아감

- **Werkzeug(WSGI 툴킷)** : 웹 서버와 파이썬 어플리케이션 사이의 통신을 담당하는 인터페이스 규격 구현
- **Jinja2(템플릿 엔진)** : 파이썬 코드를 이용해 동적인 HTML 페이지를 생성할때 사용

# 4. Flask 의 동작 방식
- **데코레이터 사용** : `@app.route('/')` 와 같은 파이썬 데코레이터를 사용하여 특정 URL 주소를 특정 함수와 연결
- **HTTP 메서드** : 기본적으로 GET 요청을 처리하며, 필요에 따라 POST, PUT, DELETE 등을 명시적으로 지정하여 처리 가능

# 5. 데이터 처리 방식
- **요청(Request) 객체** : 클라이언트가 보낸 데이터 (폼 데이터, URL 파라미터, JSON 등) 에 접근 할 때 `request` 객체를 사용
- **웅답(Response) 객체** : 처리 결과를 `jsonify` 를 통해 JSON 으로 반환하거나, `render_template`을 통해 HTML로 반환

# 6. 개발 편의 기능 
- **디버그 모드 (Debug Mode)** : 코드를 수정하면 서버가 자동으로 재시작되고, 애러 발생 시 브라우저에서 자세한 로그를 보여주는 기능 제공
