# 1. 기계 학습의 정의 
: 단순한 프로그래밍과 머신러닝의 차이점은 **'경험(Experience)'**을 통한 성능 개선에 존재

### 주요 정의
- **사무엘 (1959)**: 컴퓨터가 경험을 통해 학습하게 함으로써 세세한 프로그래밍의 번거로움을 없애는 것.
- **미첼 (1997)**: 작업 T에 대해 성능 P를 측정할 때, 경험 E를 통해 성능이 개선된다면 학습한다고 정의
- **핵심 원리**: 사례 데이터(과거 경험)를 이용해 성능 기준을 최적화하는 계산학적 방법.

# 2. 지식기반 시스템과 기계학습
: 과거에는 사람이 규칙을 직접 정해주었으나, 현대에는 데이터 중심 (기계학습)으로 주도권이 전환

### 예시: 숫자 '8'과 '단추'를 어떻게 구분할까?
- **지식기반의 한계**: "구멍이 2개고 가운데가 홀쭉하면 8이다"라고 정의해도, 실제 손글씨나 다양한 모양의 단추 데이터(심한 변화 양상)를 모두 설명하기엔 오류가 너무 많음.
- **기계학습의 해결**: 사람은 왜 그렇게 인식하는지 구체적으로 서술하지 못하더라도(암묵적 지식), 기계는 수많은 데이터를 통해 그 패턴을 스스로 찾아냄

# 3. 데이터의 분류와 용어 정리
: 머신 러닝 모델을 만들기 위해 데이터의 구조를 이해하는 것이 가장 중요

### 범주형 데이터 (Categorical Data) 
- **특징** : 이산적이며 수치로 측정이 불가능한 자료
- **예시** : 성별, 혈액형 등
> 컴퓨터 처리를 '위해 예/아니오' 방식의 테이블로 변환 하기도 함

### 수치형 데이터 (Numerical Data)
- **특징** : 수치로 측정이 가능하며 연속적이거나 이산적일 수 있는 자료
- **예시** : 키, 몸무게, 혈압 등
